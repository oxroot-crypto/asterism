## 1. 问题陈述

### 1.1 行业背景

视觉小说（Visual Novel / ADV / Galgame）是全球游戏产业中一个持续增长的细分市场。根据 SteamDB（2025 年度报告）和 VNDB（Visual Novel Database，2025 年统计数据），每年有超过 2000 部新视觉小说发布，全球市场规模估计超过 15 亿美元（来源：Visual Novel Market Report 2025, Anime News Network）。然而，创作者在工具选择上面临显著的困境。

### 1.2 当前困境

#### 困境一：引擎老化，体验落后

市场上的主流引擎多数诞生于 2000 年代初期。以 **Ren'Py**（2004年）为代表的开源引擎，虽然社区庞大、功能成熟，但其：
- 基于 Python 2/3 的脚本运行时在现代硬件上性能不佳
- 渲染管线基于 SDL2 + OpenGL，不支持现代 GPU 特性（Vulkan/DirectX 12/Metal）
- 默认 UI 审美停留在十年前的风格，需大量手写 Python 代码才能定制
- 没有官方 IDE，脚本调试和资源管理完全依赖外部工具
- Web 和移动端支持薄弱

#### 困境二：商业引擎门槛高

以 **TyranoBuilder**、**Visual Novel Maker** 为代表的商业可视化引擎虽然降低了制作门槛，但：
- 闭源且收费（$50-$100+ 授权费）
- 性能和可扩展性不足，难以支撑大型商业项目（10小时+ 游戏时长、数百个场景）
- 底层技术老化（NW.js / HTML5 Canvas），渲染效果平淡，无法实现 GPU Shader 驱动的毛玻璃模糊、粒子特效、色彩分级等现代 2D 视觉效果

#### 困境三：现代技术栈的缺失

2026 年的游戏开发者期望：
- GPU 加速的 2D 渲染（Shader 特效、粒子系统、后期处理）
- 现代开发工作流（IDE 集成的语法检查、实时预览、一键构建）
- 跨平台分发（Windows + macOS + Linux 统一支持）
- 社区驱动的开源生态（插件市场、主题共享、脚本模板）

当前没有一款引擎能同时满足以上所有期望。

### 1.3 核心痛点总结

| 痛点 | 现有方案 | 差距 |
|------|---------|------|
| 视觉品质 | Ren'Py / TyranoScript 手工定制 | 无法原生支持毛玻璃、发光、粒子等现代 UI 效果 |
| 开发效率 | 文本编辑器 + 命令行 | 缺少专业 IDE 的语法检查、实时预览、资源管理 |
| 性能 | SDL2 / HTML5 Canvas | 4K 分辨率下渲染性能不足，大场景加载慢 |
| UI 定制 | 手写代码 + PNG 素材 | 无统一主题系统，换肤成本极高 |
| 跨平台 | 各引擎支持不均 | macOS/Linux 往往是二等公民 |
| 社区生态 | 论坛 + 散落脚本 | 缺少类似 Unity Asset Store 的插件/主题市场 |

## 3. 解决方案概述

### 3.1 Asterism 是什么

**Asterism**（群星）是一个开源的、专业级 Galgame/ADV 游戏引擎 + 集成开发环境（IDE）。它提供从剧本编写到最终发布的全套工具链，目标是让创作者能制作出 GPU Shader 驱动、60fps 稳定渲染、支持 9-Slice 自适应 UI 和后期色彩分级的高品质视觉小说作品。

### 3.2 核心产品形态

```
┌──────────────────────────────────────────────────────────┐
│                    Asterism IDE                          │
│  ┌─────────┐ ┌──────────────┐ ┌──────────────┐          │
│  │ 脚本编辑器│ │  实时预览窗口  │ │  资源管理器  │          │
│  │ (Monaco) │ │ (GPU Render) │ │ (拖拽导入)   │          │
│  └─────────┘ └──────────────┘ └──────────────┘          │
│  ┌─────────┐ ┌──────────────┐ ┌──────────────┐          │
│  │ 分支流程图│ │  角色管理器   │ │  主题编辑器  │          │
│  │ (VueFlow)│ │ (Sprite Grid)│ │ (可视化配置) │          │
│  └─────────┘ └──────────────┘ └──────────────┘          │
│                                                         │
│              一键构建 → 独立游戏安装包                      │
├──────────────────────────────────────────────────────────┤
```

### 3.3 关键设计决策

| 决策 | 选择 | 理由 |
|------|------|------|
| 引擎语言 | Rust | 极致性能 + 内存安全 + 跨平台编译，适合 GPU 密集型 2D 渲染 |
| IDE 技术 | Tauri v2 + Vue 3 | 轻量原生桌面应用（~5MB），前端灵活，后端直接复用引擎核心 |
| 脚本语言 | 自定义 .aster DSL | 领域优化语法（自然易读），支持静态分析和编译优化 |
| 渲染后端 | wgpu | 统一 Vulkan/DX12/Metal 抽象，为未来 WebGPU 铺路 |
| 音频后端 | kira | Rust 生态最佳音频框架，Tweening + 多混音组 |
| UI 系统 | GPU Shader 驱动 | 原生支持毛玻璃模糊/渐变/发光/粒子，9-Slice 自适应缩放，60fps UI 动画 |
| 许可协议 | MIT | 最大化社区采用和生态发展 |

## 4. 核心价值主张

### 4.1 与竞品的差异化优势

| 维度 | Asterism | Ren'Py | TyranoBuilder | WebGAL | Visual Novel Maker |
|------|----------|--------|---------------|--------|-------------------|
| **开源** | ✅ MIT | ✅ MIT | ❌ 闭源收费 | ✅ MPL | ❌ 闭源收费 |
| **专业IDE** | ✅ Tauri原生 | ❌ 无官方IDE | ✅ 可视化 | ❌ Web | ✅ 可视化 |
| **GPU渲染** | ✅ wgpu (Vulkan/DX12/Metal) | ⚠️ OpenGL 2.1 / 部分 Shader 支持 | ❌ HTML5 Canvas | ❌ WebGL | ❌ NW.js |
| **现代UI效果** | ✅ Shader级毛玻璃/发光/粒子 | ⚠️ 需手写扩展 | ❌ 有限 | ❌ 有限 | ❌ 有限 |
| **主题系统** | ✅ 声明式theme.toml + 9-Slice | ⚠️ 需Python编码 | ⚠️ 有限 | ⚠️ 有限 | ⚠️ 有限 |
| **脚本语言** | ✅ 自定义DSL（领域优化） | ✅ Python | ⚠️ 可视化为主 | ✅ 类JSON | ⚠️ 可视化为主 |
| **静态分析** | ✅ 编译期校验 | ❌ 运行时错误 | ❌ 无 | ❌ 无 | ❌ 无 |
| **跨平台桌面** | ✅ Win/Mac/Linux | ✅ Win/Mac/Linux | ✅ Win/Mac | ✅ Web | ✅ Win/Mac |
| **性能(4K)** | ✅ 原生GPU加速 | ⚠️ 一般 | ❌ 较差 | ❌ 较差 | ❌ 较差 |
| **社区生态** | 🚧 建设中 | ✅ 成熟 | ❌ 有限 | 🚧 年轻 | ❌ 有限 |

### 4.2 四大核心价值主张

1. **商业级视觉品质** — 引擎内置基于 GPU Shader 的 UI 渲染管线，原生支持毛玻璃模糊、渐变、发光、粒子、色彩分级等现代视觉效果。默认主题提供完整的日式 ADV 美学风格，涵盖 12 种游戏内 UI 界面的统一视觉设计。

2. **声明式创作体验** — 自定义 `.aster` DSL 脚本语言，语法贴近自然故事描述。创作者只需表达"要什么"，不需关心"怎么实现"。`theme.toml` 让 UI 定制从代码变为配置。

3. **一体化开发工作流** — 从剧本编写、资源管理、实时预览到一键多渠道发布，全部集成在一个轻量桌面应用中。无需在 3-4 个工具之间切换。

4. **开源与社区驱动** — MIT 许可，完全免费。插件系统和主题市场让社区持续贡献价值，避免闭源引擎"厂商停滞、用户被困"的风险。

## 6. 范围与边界

### 6.1 项目包含（Scope In）

- ✅ 桌面优先的 Galgame/ADV 运行时引擎（Windows + macOS + Linux）
- ✅ 集成开发环境（IDE）：脚本编辑、资源管理、实时预览、一键构建
- ✅ 自定义脚本语言 `.aster`：声明式、领域优化、可静态分析
- ✅ GPU 驱动的 2D 渲染管线：Sprite、文本、UI 控件、转场特效
- ✅ 音频系统：BGM / SE / Voice 多轨混音
- ✅ UI 主题系统：默认精美主题 + 声明式自定义（theme.toml + 9-Slice PNG）
- ✅ 存档系统：多槽位、缩略图、快照式恢复
- ✅ 项目打包工具：一键生成 Windows/macOS/Linux 安装包
- ✅ 完整的中文文档和 CLlAUDE.md 行为规范

### 6.2 项目不包含（Scope Out — MVP 阶段）

- ❌ 移动端原生 App（后续迭代）
- ❌ Web 端浏览器运行（后续迭代，寄望于 WebGPU）
- ❌ 可视化无代码场景编辑器（v1.0.0）
- ❌ Live2D / Cubism 骨骼动画（v1.0.0）
- ❌ 视频播放（OP/ED 动画，v1.0.0）
- ❌ 游戏内插件系统（WASM，v1.0.0）
- ❌ Steam 集成（成就/云存档，v1.0.0）
- ❌ 多语言/i18n 框架（v1.0.0）
- ❌ 内置素材商店/社区市场（长期愿景）
- ❌ 3D 渲染能力（不在产品愿景内）
- ❌ 在线协作文档编辑（不在产品愿景内）

### 6.3 技术边界

- **仅 2D**：引擎专注于 2D 视觉小说渲染，不引入 3D 渲染管线
- **桌面优先**：MVP 仅支持桌面平台，但架构预留 Web/移动端扩展点
- **不绑定平台**：不对特定平台（如 Steam、itch.io）做排他性集成
- **不提供 DRM**：引擎本身开源，不对产出的游戏提供数字版权保护（创作者可自行对接第三方 DRM）

## 7. 项目理念

> **"让每一颗故事之星都能闪耀。"**

Asterism（群星）的命名源于一个信念：每一位创作者都有一个独特的故事，而每个故事都像一颗星辰。我们的使命是提供最好的工具，让这些星辰汇聚成璀璨的星空。

作为纯 Vibe Coding 开源项目，Asterism 的所有代码由 AI 独立完成，严格遵循 `.claude/CLAUDE.md` 中定义的编码规范和行为准则。我们相信，AI 驱动的开发模式能够以极高的效率和一致性，为视觉小说社区打造一款真正现代化的创作工具。
