## 1. 需求编号体系

| 编号前缀 | 类别 | 说明 |
|---------|------|------|
| `REQ-ENG-` | 引擎功能需求 | 运行时引擎必须具备的功能 |
| `REQ-IDE-` | IDE功能需求 | 集成开发环境必须具备的功能 |
| `REQ-DSL-` | 脚本语言需求 | .aster DSL 语言的语法和能力需求 |
| `REQ-UI-` | UI/主题需求 | 游戏 UI 的主题、渲染和交互需求 |
| `NFR-PERF-` | 性能需求 | 响应时间、帧率、加载速度等 |
| `NFR-SEC-` | 安全需求 | 认证、授权、数据保护 |
| `NFR-USAB-` | 可用性需求 | 易用性、可访问性、国际化基础 |
| `NFR-MAIN-` | 可维护性需求 | 代码规范、测试覆盖、文档要求 |
| `NFR-COMPAT-` | 兼容性需求 | 跨平台、分辨率适配 |

## 2. 功能需求

### 2.1 P0 — MVP 必备功能（v0.1~v0.3 交付）

> **优先级定义**：P0 功能是项目的核心骨架。缺了任何一个，引擎无法成为一个可用的视觉小说播放器，IDE 无法完成从创建到导出的基本工作流。

#### 2.1.1 引擎核心 — 脚本与执行

| 编号 | 名称 | 描述 | 验收标准 | 优先级 |
|------|------|------|---------|--------|
| REQ-ENG-001 | DSL 脚本解析 | 引擎必须能够解析 `.aster` 格式的脚本文件，该 DSL 为领域优化语法，支持场景定义、对话、选择支、跳转等视觉小说核心语义 | 1）给定合法的 `.aster` 文件，解析器返回完整 AST 无错误；2）给定非法语法，解析器返回带行号/列的诊断信息；3）解析性能：10,000 行脚本 < 100ms | P0 |
| REQ-ENG-002 | 字节码编译与执行 | 引擎必须将 AST 编译为字节码，并由指令式虚拟机执行。VM 支持顺序执行、条件跳转、变量/旗标存取、子例程调用 | 1）相同脚本经编译→执行产生的结果一致；2）VM 执行时帧时间 < 0.5ms；3）变量/旗标的读写符合预期语义 | P0 |
| REQ-ENG-003 | 变量与旗标系统 | 支持整型、浮点、字符串、布尔四种值类型的变量存储。支持旗标（命名布尔值）的 set/unset/toggle/check 操作 | 1）变量在场景间保持有效；2）旗标操作结果正确；3）支持 `if/elif/else` 条件分支 | P0 |

| REQ-ENG-011 | 背景图片渲染 | 加载并显示背景图片（PNG/WebP 格式），自动缩放适配窗口分辨率（保持宽高比，裁剪或留黑边） | 1）背景正确显示在画面最底层；2）切换背景时上一背景被正确替换；3）支持多种宽高比的素材 | P0 |
| REQ-ENG-012 | 角色立绘渲染 | 加载并显示角色立绘精灵（PNG/WebP，含透明度），支持多个立绘同时显示，每个立绘可独立设置位置（左/中/右/自定义坐标）和透明度 | 1）角色立绘正确显示在背景之上；2）多角色同时显示时层级正确；3）立绘位置和透明度可正确设置 | P0 |
| REQ-ENG-013 | 对话文本渲染 | 在画面指定区域渲染对话文本，含说话者名字。支持基本字符集（CJK + 拉丁 + 常见符号）。文字支持基本颜色和透明度设置 | 1）文本在文本框区域正确显示；2）CJK 字符无乱码；3）说话者名字与正文视觉区分 | P0 |
| REQ-ENG-014 | 文字逐字显示（打字机效果） | 正文以逐字方式显示（类似打字机效果），速度可配置。玩家点击/按键可跳过动画直接显示全部文本 | 1）字符逐字出现，速度可调；2）点击后立即显示全部剩余文本；3）逐字显示时音频同步不受影响 | P0 |

| REQ-ENG-023 | 场景/脚本跳转 | 支持 `goto` / `jump` 语义，从当前脚本位置跳转到指定场景的指定标签 | 1）跳转后从目标标签开始执行；2）跨文件跳转正确；3）跳转不导致变量丢失 | P0 |

### 2.2 P1 — 核心体验功能（v0.4~v0.5 交付）

> **优先级定义**：P1 功能是区分"可用的引擎"和"专业的引擎"的分水岭。它们不阻碍 MVP 发布，但缺少则引擎不具备商业级视觉小说的制作能力。

#### 2.2.1 DSL 语法增强

| 编号 | 名称 | 描述 | 验收标准 | 优先级 |
|------|------|------|---------|--------|
| REQ-DSL-001 | 宏定义与调用 | 支持 `macro` 语法定义可复用脚本片段（如"摇头"动作序列），在场景中通过 `@macro_name` 调用 | 1）宏内参数正确替换；2）宏内语法错误可追溯到调用位置 | P1 |
| REQ-DSL-002 | 模板/继承 | 支持场景模板（如"标准对话场景"），新场景可继承模板并覆盖子元素 | 1）子场景能继承模板的背景/音乐/角色默认配置；2）覆盖的字段生效 | P1 |
| REQ-DSL-003 | 表达式增强 | 支持算术运算、字符串拼接、三元运算符、数组和映射字面量 | 1）表达式求值结果正确；2）类型不兼容时报错 | P1 |

| REQ-ENG-052 | 文字特效 | 支持打字机效果外的文字展现特效：抖动（shake）、彩虹渐变（rainbow）、上标注音（ruby/furigana）、粗体/斜体 | 1）每种文字特效视觉正确；2）特效可叠加使用；3）对帧率影响 < 5% | P1 |

| REQ-ENG-077 | 音乐鉴赏 | 展示游戏中所有 BGM 曲目列表，可播放/暂停、显示曲目标题和时长 | 1）曲目列表正确；2）播放正常且支持进度拖拽 | P1 |
